‚≠ê Best Practices

üéØ File Organization

  • Consistent naming: Use "Artist - Album - Track.mp3" format
  • Track numbers: Include "01 - Song Title.mp3" for proper ordering
  • Folder structure: Artist ‚Üí Album ‚Üí Tracks hierarchy
  • Metadata: Fill all ID3 tags (artist, title, album, year, genre)

‚ö° Processing Strategy

  • Start small: Test with 5-10 songs before large batches
  • Batch processing: Process 50-100 songs at a time
  • Monitor resources: Close other apps during processing
  • Regular saves: System auto-saves every 5 tracks

üìä Quality Control

  • Review transcriptions: Check accuracy for important songs
  • Manual corrections: Edit lyrics files if needed
  • Re-transcribe: Use "Force Reprocess" for better quality
  • Backup regularly: Export catalog after major updates

üîÑ Workflow Optimization

  • Peak hours: Process large libraries during off-peak times
  • SSD storage: Use solid-state drives for faster processing
  • Network files: Copy files locally for faster access
  • Parallel processing: System handles multiple files efficiently

üí° Pro Tips

  • Use consistent artist names (avoid "The Beatles" vs "Beatles")
  • Include release years for better cataloging and analytics
  • Add genre tags for better organization and filtering
  • Keep original files backed up before processing
  • Integrate with ICETAP for album art annotation
